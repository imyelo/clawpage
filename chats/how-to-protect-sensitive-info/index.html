Claude09:03

Here's the typical flow:

  1. Run /clawpage in your conversation — the skill exports and attempts to redact
  2. Create a PR to your Chats Share repository — this lets you review changes before merging
  3. Manually review the YAML file in the PR — check for anything missed
  4. Manually add REDACTED to anything that looks sensitive
  5. Merge the PR — the site auto-deploys and your chat is live

Critical: Make sure your repository is private. If it's public, anyone can view the raw YAML file at your GitHub repository history and see the original unredacted data.

The rendered page (/share/your-session) is safe — only the YAML source contains the original values.
